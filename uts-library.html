项目背景

悉尼科技大学图书馆 Building 1 是校园内最繁忙的学习与研究场所，日均访客超过 2,000 人次。传统的人工巡查方式难以实时掌握各楼层座位占用情况、分区状态及图书资源动态，导致场馆运营效率受限。

本项目由导师研究团队发起，目标是构建一套实时可视化运营大屏，将多类传感器与业务系统的数据流汇聚呈现，辅助馆员做出快速、准确的运营决策。

我的职责

  • 独立完成大屏全部 UI/UX 设计，从零搭建视觉规范
  • 实现六楼层实时座位占用模块，含进度条热力渐变逻辑
  • 设计分区状态看板（静音区、自习室、机房、协作区）
  • 构建客流量时段对比柱状图，支持今日 vs 昨日叠加对比
  • 实现 KPI 指标卡自动刷新与告警面板优先级排序展示
  • 完成多模块响应式布局，适配不同尺寸大屏终端

核心设计决策

信息层级设计：将数据按「总览 KPI → 楼层细分 → 分区详情 → 时段趋势 → 告警」五层递进排布，确保运营人员在 5 秒内完成关键信息的捕获，无需学习成本。

颜色语义编码：统一定义三色占用状态（红色 ≥80% 近满 / 黄色 50–79% 较忙 / 绿色 <50% 空闲），与告警面板颜色体系保持一致，降低认知负担。

实时性优先：所有占用数据模块均采用轮询刷新机制设计，KPI 卡片支持秒级更新，告警面板按时间戳倒序展示，确保运营人员始终看到最新状态。

暗色主题选型：考虑到大屏通常部署于馆内公共展示区，暗色背景在低光环境下对比度更高，减少视觉疲劳，同时在视觉上凸显数据本身。

技术栈

HTML5 CSS3 Grid / Flexbox JavaScript ES6+ 实时数据渲染 响应式布局 多模块 UI 组件 颜色语义系统
